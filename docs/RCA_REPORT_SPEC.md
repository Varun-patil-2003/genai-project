# RCA Report Specification — Module C

This document defines the structure of the auto-generated PDF RCA reports.
`utils/pdf_reporter.py` implements this spec using `reportlab.platypus`.

---

## Trigger

A report is generated when a ticket transitions to `status = "Resolved"`.

The orchestration layer (`orchestration/rca_orchestrator.py`) pulls:
1. The ticket record from the repository
2. Logs from the `ANOMALY_LOOKBACK_MINUTES` window around the incident timestamp
3. Feeds both to the LLM with the RCA prompt template (`prompts/rca/rca_template.jinja2`)
4. Passes the structured LLM output to `utils/pdf_reporter.py`

Output path: `reports/{ticket_id}_rca_{timestamp}.pdf`

---

## Report sections

### Cover page
- Report title: "Root Cause Analysis Report"
- Ticket ID and title
- System/component affected (e.g. "London SBC — SIP Registration")
- Severity: Critical / High / Medium
- Generated by: NetOps AI Sentinel
- Date generated

### 1. Executive summary
One paragraph. Non-technical language. States what failed, when, and what fixed it.

### 2. Incident timeline
Table with columns: `Timestamp | Event | Source`.
Populated from log entries in the incident window.

### 3. Root cause
Subsections:
- **Primary cause**: the direct technical cause
- **Contributing factors**: environmental or configuration factors
- **Detection gap**: why monitoring didn't catch this sooner

### 4. Impact assessment
- Services affected
- Duration of impact
- User/device count affected (if available)

### 5. Resolution
- Steps taken to resolve
- Who resolved it (from ticket assignee field)
- Time to resolution

### 6. Prevention recommendations
Bullet list of actions to prevent recurrence. LLM-generated based on root cause.

### 7. Appendix — raw logs
Truncated log excerpt (max 50 lines) from the incident window.
Monospace font.

## PDF styling (reportlab)

```python
from reportlab.lib.pagesizes import A4
from reportlab.platypus import SimpleDocTemplate, Paragraph, Table, Spacer, PageBreak
from reportlab.lib.styles import getSampleStyleSheet, ParagraphStyle
from reportlab.lib import colors

# Page setup
doc = SimpleDocTemplate(output_path, pagesize=A4, topMargin=40, bottomMargin=40)

# Color palette
PRIMARY_COLOR = colors.HexColor("#1a3a5c")   # Dark navy — headers
ACCENT_COLOR  = colors.HexColor("#e84855")   # Red — severity badge
LIGHT_GRAY    = colors.HexColor("#f5f5f5")   # Table row alternating fill
```

Use `SimpleDocTemplate` with `Platypus` flowables — not `canvas` directly.
This allows page breaks, paragraph reflow, and table pagination to work automatically.

## LLM output schema (`prompts/rca/rca_template.jinja2` response format)

The LLM must return JSON matching this schema. Enforce via system prompt:

```json
{
  "executive_summary": "string",
  "root_cause_primary": "string",
  "root_cause_contributing": ["string"],
  "detection_gap": "string",
  "impact_services": ["string"],
  "impact_duration_minutes": 42,
  "resolution_steps": ["string"],
  "prevention_recommendations": ["string"]
}
```
